# 序列化

Jackson（JSON）序列化与Java原生序列化区别及适用场景总结：

区别：

1. Jackson序列化为JSON文本，易读、跨语言；原生序列化为二进制字节流，仅限Java。
2. Jackson无需实现Serializable，原生必须实现。
3. Jackson丢失类型信息，原生保留完整类型。
4. Jackson可自定义序列化，原生扩展性较差。

适用场景：

- Jackson：Web接口、前后端通信、配置文件、跨语言数据交换。
- 原生：Java应用内部对象存储、缓存、RMI、只在Java环境下使用。

# 自动注入

IDE 提示不推荐使用自动注入（如字段上的 `@Autowired`），主要原因有：

1. **可测试性差**：字段注入让单元测试变得困难，因为无法通过构造器直接传递依赖。
2. **不可变性差**：字段注入通常不能配合 `final`，依赖可以被修改，降低了代码的健壮性。
3. **依赖不明确**：构造器注入让依赖一目了然，字段注入则隐藏了依赖关系。
4. **易于维护**：构造器注入更容易发现缺失依赖，编译时即可发现问题。

因此，推荐使用构造器注入，提升代码质量和可维护性。


# url编码

Base64 和 Base62 编码的主要区别如下：

- **字符集不同**：  
  - Base64 使用 64 个字符：`A-Z`, `a-z`, `0-9`, `+`, `/`（有时 URL 安全版用 `-` 和 `_`）。  
  - Base62 只用 62 个字符：`A-Z`, `a-z`, `0-9`，没有特殊符号。

- **编码长度**：  
  - Base64 编码后字符串更短，但可能包含 `+`、`/`、`=` 等特殊字符。  
  - Base62 编码长度略长，但只包含字母和数字，更适合 URL、文件名等场景。

- **用途**：  
  - Base64 常用于二进制数据转文本（如图片、加密数据传输）。  
  - Base62 常用于生成短链接、邀请码等需要纯字母数字的场景。

- **兼容性**：  
  - Base62 更适合需要避免特殊字符的系统或 URL。

总结：Base62 更适合需要“纯字母数字”的编码场景，Base64 更适合高效传输和存储二进制数据。
