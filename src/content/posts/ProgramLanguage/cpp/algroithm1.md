### 动态规划

之前对动态规划问题的逻辑结构不了解，不理解为什么总是第一维度不考虑同阶段，然后抽象模型也总是平面，到了高纬度动态对话就难以构建抽象结构。

在阶段型动态规划中，**阶段**通常被设为第一维，表示问题求解的线性进程（如时间、步骤、物品编号）。状态转移可以依赖**上一阶段**，也可能依赖**同一阶段**，具体取决于问题的决策特性。下面从定义、转移类型和典型例子三个方面进行总结。

#### 阶段型DP的定义

- **阶段**：具有明确顺序的进程变量，如物品编号 `i`、天数 `t`、步数 `k` 等。通常作为第一维。
- **状态**：`dp[i][...]` 表示在第 `i` 阶段，结合其他维度（如容量、余额）描述的子问题最优值。
- **计算顺序**：按阶段递增（或递减）进行，保证无后效性。

#### 转移的两种类型

##### 依赖上一阶段
这是最常见的阶段型转移。当前阶段的状态只由**上一阶段**的状态通过决策得到，不允许在同一阶段内重复利用当前阶段的决策。 
**典型例子**：

- **0-1背包**：`dp[i][j] = max(dp[i-1][j], dp[i-1][j-w[i]] + v[i])`，因为每个物品只能选一次。
- **最长上升子序列（朴素）**：`dp[i] = max(dp[j] + 1)`（`j < i`），这里 `j` 是更早的阶段。
- **数字三角形**：`dp[i][j] = max(dp[i-1][j-1], dp[i-1][j]) + a[i][j]`。

**特点**：阶段之间界限清晰，每个阶段的状态仅由前一阶段推导，通常可以用滚动数组优化空间。

##### 依赖同一阶段

某些问题允许在当前阶段内进行多次决策，或者需要利用同一阶段内已计算的较小规模子问题。此时，转移可能引用**同一阶段**的其他状态。  
**典型例子**：
- **完全背包**（二维形式）：`dp[i][j] = max(dp[i-1][j], dp[i][j-w[i]] + v[i])`。其中 `dp[i][j-w[i]]` 是同一阶段（第 `i` 个物品）且容量更小的状态，通过按容量正序计算，实现了物品的重复选取。
- **多重背包的单调队列优化**：在阶段内利用队列维护同一阶段的候选值。
- **某些线性DP**：如“最大子段和”的在线算法，虽然不显式分阶段，但本质上是在同一阶段内更新当前最优。

**特点**：同一阶段内的转移通常需要特定的遍历顺序（如容量从小到大），以确保被依赖的状态已经计算完毕。

#### 与结构型DP的对比
- **结构型DP**（如区间DP、树形DP）中，第一维往往不是线性阶段，而是空间位置。转移可能从同一维度的不同值（如不同的左端点）获取信息，例如区间DP的 `dp[l][r] = max(dp[l][k] + dp[k+1][r] + cost)`，这里 `l` 和 `k+1` 都是第一维的不同取值，属于“同级别比较”，而非阶段型中的“同一阶段内横向依赖”。
- 阶段型中的同一阶段转移，第一维的值保持不变，只是其他维度变化；而结构型中的同级别比较，第一维本身发生了变化。

#### 结构性DP

从小到大逆向推导

#### 总结
- **阶段型DP的核心**：阶段作为线性进程，状态按阶段顺序计算。
- **转移方向**：
  - 若决策不允许在当前阶段重复进行，则依赖**上一阶段**（如0-1背包）。
  - 若允许重复或需要利用当前阶段的子问题，则可能依赖**同一阶段**（如完全背包）。
- **判断依据**：关键在于问题的决策是否具有“重复性”以及子问题的定义方式。
- **实现技巧**：依赖同一阶段时，需注意遍历顺序（如容量正序、区间长度递增）以保证依赖的子问题已求解。

理解这些区别有助于正确设计状态转移方程，并选择合适的优化方法。

### 过河卒

有马所在中心所有可以到达的位置都不可以经过，棋盘之中只能去往右下角，动态数组基础版本。

#### 代码

```cpp
#include <iostream>
using namespace std;

const int dx[8] = {1, 2, 2, 1, -1, -2, -2, -1};
const int dy[8] = {2, 1, -1, -2, -2, -1, 1, 2};

int main() {
    int n, m, x, y;
    cin >> n >> m >> x >> y;

    bool block[21][21] = {false};
    block[x][y] = true;
    for (int k = 0; k < 8; ++k) {
        int nx = x + dx[k], ny = y + dy[k];
        if (nx >= 0 && nx <= 20 && ny >= 0 && ny <= 20)
            block[nx][ny] = true;
    }

    long long dp[21][21] = {0};
    dp[0][0] = block[0][0] ? 0 : 1;

    for (int i = 0; i <= n; ++i) {
        for (int j = 0; j <= m; ++j) {
            if (block[i][j]) {
                dp[i][j] = 0;
                continue;
            }
            if (i > 0) dp[i][j] += dp[i-1][j];
            if (j > 0) dp[i][j] += dp[i][j-1];
        }
    }
    cout << dp[n][m] << endl;
    return 0;
}
```

### 方格取数

这是不同于二维动态规划问题，二维可以直观的从数组中获取状态机，但是高维就不可以，只能基于上一个状态和所需信息进行动态优化。

在动态规划中，维度的数量取决于描述一个子问题所需要的独立变量的个数。二维DP通常用两个变量（如位置、容量）表示状态，而三维、四维DP则引入更多变量以处理更复杂的约束。下面结合你提供的“方格取数”代码，具体说明三维、四维DP的定义及其与二维解题思路的区别。

#### 二维DP的典型特征
- **状态定义**：通常由两个变量唯一确定一个子问题。例如，在单次路径问题中，`dp[i][j]` 表示从起点走到 `(i,j)` 能获得的最大数字和。
- **转移方程**：只考虑当前步的决策，如从上方或左方转移过来。
- **适用范围**：问题只有一个独立的进程或路径。

#### 三维DP的引入
当问题需要同时考虑两个相互影响的进程时，可能需要增加维度。例如，在“方格取数”问题中，如果要求走两次且两次路径可以交叉，但同一个格子只能取一次数，那么单靠二维无法同时记录两条路径的状态。此时就需要将两条路径的当前位置都作为状态变量，从而形成四维（两个二维坐标组合成四维）。但有时如果两个进程有某种对称性，可能可以压缩到三维，比如当两条路径步数相同时，可以用步数和两个横坐标表示（因为纵坐标可由步数减去横坐标得到）。但最直接的方式是四维。

#### 四维DP的定义——以代码为例
你提供的代码正是用四维记忆化搜索解决“方格取数”问题。其状态定义为：
```cpp
f[x][y][x2][y2]
```
表示**第一条路径走到 `(x, y)`，第二条路径走到 `(x2, y2)` 时，从这两个位置继续走到终点 `(N, N)` 能获得的最大数字和**。这里四个坐标就是四个独立变量，因此是四维DP。

**为什么需要四维？**
- 两条路径是同时进行的，每一步两条路径都可以选择向下或向右。
- 格子上的数只能被取一次，如果两条路径在同一时刻走到同一个格子，则只加一次该格子的数。
- 为了判断是否重复，必须同时知道两条路径的当前位置，因此需要四个坐标。

**状态转移**：
代码中考虑了四种移动组合（都向下、一向下另一向右、一向右另一向下、都向右），并加上新到达格子的数值，若重合则减去一份。这相当于在四维空间中进行递推。

#### 与二维解题思路的区别
1. **状态变量的选择**：
   - 二维：通常由问题自然引出两个变量（如行和列）。
   - 三维/四维：需要更细致地分析问题中涉及的独立因素。例如在方格取数中，两条路径的当前位置都是必须记录的，因此需要四个变量。有时可以通过数学关系减少一维（如利用总步数），但本质仍是高维。

2. **转移的复杂度**：
   - 二维：转移方向通常固定（如2个方向），时间复杂度为 O(n²)。
   - 四维：每一步有 2×2=4 种可能的移动组合，但状态数变为 O(N⁴)，时间复杂度可能高达 O(N⁴)（若枚举所有状态）。不过通过记忆化搜索，实际只计算必要的状态。

3. **空间消耗**：
   - 二维：数组大小为 O(N²)。
   - 四维：数组大小为 O(N⁴)，当 N 较大时可能无法承受。因此高维DP往往需要优化（如滚动数组、状态压缩）或利用问题特性降维。

4. **处理重复计数**：
   - 在二维单路径问题中，每个格子只经过一次，无需考虑重复。
   - 在多路径问题中，需要额外处理重合情况，如代码中用 `-s[x+1][y]*(x+1==x2+1&&y==y2)` 来避免重复加同一个格子。

#### 总结
三维、四维DP是二维DP的自然推广，它们通过增加状态变量来描述更复杂的场景。解题时，关键是找出所有必要的独立变量，并设计正确的状态转移。虽然高维DP在理论上思路类似，但实际实现中需注意时空复杂度，并思考是否可以通过降维技巧（如利用对称性、合并变量）来简化问题。你提供的代码正是四维DP的一个典型实例，清晰地展示了如何用记忆化搜索处理两条路径的同步移动问题。



### 矩阵取数

#### 关键点

1. 容易调入贪心陷阱，每次都选最小容易让中间值失去获得更大权重的可能，应该采取动态规划。

2. 其为区间动态规划，从最小区间逐渐左右拓展迭代到最长的区间最大得分可能。

3. 不同行互不影响，其实就是每一行独立进行动态规划计算，最后简单相加即可。

4. 需要从小到大处理$2^i$的权重问题，其思路是每一次向外拓展都乘以2，这样其实倒推出来就是最开始的小区间权重最高。

5. 处理数值范围问题

   

#### 

#### 实现思路



```cpp
#include <bits/stdc++.h>
using namespace std;
int n, m, a[90][90];
__int128 f[90][90], ans;

void out (__int128 x) {
    if(x>9) out(x/10);
    putchar(x%10+'0');
}

int main () {
    cin >> n >> m;
    for (int i=1; i<=n; ++i)
        for (int j=1; j<=m; ++j)
            cin >> a[i][j];

    for (int i=1; i<=n; ans+=f[1][m], memset(f, 0, sizeof f), ++i) 
        for (int len=1; len<=m; ++len) 
            for (int l=1, r=l+len-1; r<=m; ++l, ++r)
                f[l][r]=max(f[l+1][r]+a[i][l], f[l][r-1]+a[i][r])*2;

    out(ans);
    return 0;
}
```

1. 单独构建一个out函数，用于输出结果，默认c++的整数输出输入流不支持大于128位的整数操作，out函数将数字x的最低位（个位）转换为字符并输出。

   详细解释：

   - x%10：取x的个位数字（例如x=123，x%10=3）。
   - +'0'：将数字3转换为字符'3'，因为字符'0'的ASCII码是48，数字加上'0'就变成对应的字符。
   - putchar(...)：输出这个字符到屏幕。

   所以，putchar(x%10+'0');就是输出x的个位数字对应的字符。常用于大整数递归输出每一位。

2. 以后实际问题的范围从数组下标1开始即可，这样与实际序号对应，降低心智负担，并且可以预留[1-0]这样的边界情况为边界值直接处理；同样在一开始的非法区间`f[1][0]`也会使用。

3. 使用的动态规划数组为二维数组，加上不同行区间就是三维，二维数组有一半默认值为空，应对不合法空区间。

```cpp
for (int i = 1; i <= n; ans += f[1][m], memset(f, 0, sizeof f), ++i) 
    for (int len = 1; len <= m; ++len) 
        for (int l = 1, r = l + len - 1; r <= m; ++l, ++r)
            f[l][r] = max(f[l+1][r] + a[i][l], f[l][r-1] + a[i][r]) * 2;
```

1. 外层循环：依次处理每一行，每行结束后将该行的最优值累加到 `ans`，并清空 `f` 准备下一行。
2. 内层两层循环：完成该行的区间 DP。
3. 最后输出 `ans`。

三层，一层行数，二层区间长度，三层逐步移动区间并乘以二，每一层穷举当前层所有情况，但是最终迭代后过去层只记录最优结果

### 三连击



1. 没有太多巧妙方法，核心是遍历（1-1000/3）a的值
2. 如何判断是否1-9，就是转为字符串合并然后排序`s.begin(), s.end()`这样就可以判断是否满足“123456789”
3. 只有遍历到符合要求的才输出。

```cpp
// 寻找三位数 1:2:3
// 遍历所有a的组合，然后计算b和c，字符串排序看看是否正好为1-9
int main() {

    for (int a = 123; a <= 329; ++a) { // a最小123，最大329（因为3a最大987）
        int b = a * 2;
        int c = a * 3;
        string s = to_string(a) + to_string(b) + to_string(c);
        if (s.length() != 9) continue;
        sort(s.begin(), s.end());
        if (s == "123456789") {
            cout << a << " " << b << " " << c << endl;
        }
    }
    return 0;
}
```







### 反转链表

在进行链表/数组操作的时候要注意消除边界情况。

给定一段输入，构建链表，并且翻转第i到第j个节点，并输出反转后的链表
